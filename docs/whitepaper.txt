Executive SummaryAegisMesh is a privacy-first, autonomous collaboration platform designed to orchestrate specialized AI agents within local development environments. By implementing a Service Mesh for Agents, AegisMesh solves the "Cognitive Ceiling" problem of single-agent systems through a decentralized Agent-to-Agent (A2A) protocol and a verifiable Evidence Graph. The system is engineered for high-availability diagnostics on consumer-grade hardware (16GB RAM / i5 CPU), focusing on reliability, observability, and deterministic safety.2. System Architecture LayersAegisMesh adopts a layered architecture to ensure separation of concerns and system resilience.2.1 The Gateway Layer (Entry Point)The Gateway acts as the secure interface between the user and the autonomous mesh.Responsibilities: Request normalization, Authentication, and Trace Initialization.Implementation: A lightweight FastAPI service that converts natural language intent into a standardized internal investigation schema.2.2 The Supervisor Engine (Orchestration)The Supervisor is the central reasoning hub of the mesh. It manages the investigation lifecycle using a LangGraph state machine.Task Decomposition: Breaking high-level intent into atomic diagnostic sub-tasks.Agent Selection: Querying the Registry for specialists based on skill-match and confidence scores.Parallel Orchestration: Managing concurrent A2A RPC calls and state synchronization.2.3 The Control Plane (AegisRegistry)Following RFC 8615, the Registry manages the lifecycle of the mesh workforce.Discovery: Agents announce presence via JSON-RPC over HTTP.High Availability (HA): To mitigate the Registry as a single point of failure, the Supervisor maintains a Local Agent Cache of the last-known healthy directory.3. 4. Security & Safety ModelAegisMesh operates on a Non-Destructive Execution principle to protect the developer's local environment.Read-Only Boundary: Agents are strictly prohibited from executing destructive commands (e.g., rm, taskkill). The system generates remediation recommendations rather than autonomous fixes.Scope Restriction: Filesystem access for diagnostic agents is restricted to the specific project directory identified in the investigation trace.5. Resilience & Failure Mode AnalysisFailure TypeSystem ResponseAgent TimeoutTrigger exponential backoff (Max 3 retries).Agent CrashRegistry marks agent unhealthy; Supervisor selects fallback.Registry UnavailableSupervisor falls back to In-Memory Agent Cache.Memory PressureGovernor forces a switch from parallel to sequential execution.6. Observability & Distributed TracingAegisMesh implements OpenTelemetry (OTel) standards to ensure the mesh is transparent.Trace Propagation: A unique trace_id is generated at the Gateway and injected into every downstream RPC call.Telemetry: The system monitors agent_latency, task_success_rate, and diagnosis_duration.7. EvidenceGraph Reasoning EngineAegisMesh builds a Directed Acyclic Graph (DAG) of findings (backed by NetworkX + SQLite).Nodes: Facts (e.g., "CPU Spike @ 12:04").Edges: Logical relationships (e.g., "CAUSED_BY").Causal Inference: The Supervisor traverses the graph to identify the highest-confidence causal chain, ensuring Explainable AI (XAI).8.
